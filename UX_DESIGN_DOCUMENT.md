## 1. 전체 UX 흐름 요약

### 1.1 하단 탭 구조 (4개 고정)
```
┌─────────┬─────────┬─────────┬─────────┐
│   홈    │ 대화분석 │  답장   │   MY   │
└─────────┴─────────┴─────────┴─────────┘
```

### 1.2 사용자 여정 맵

```
[앱 진입]
    ↓
[홈 탭] ← 매일 첫 진입점
    ├─ "오늘, 이대로 가도 될까?" (감정적 질문)
    ├─ 오늘의 연애 상태 프리뷰
    └─ CTA: "오늘의 연애 상세 보기" → 연애 상세 페이지
         ↓
    [상세 페이지에서]
    ├─ "대화분석이 필요해" → 대화분석 탭
    └─ "답장이 필요해" → 답장 탭
         ↓
[대화분석 탭]
    ├─ 입력 → 분석 → 인사이트
    └─ 자연스러운 연결: "답장이 필요하면?" → 답장 탭
         ↓
[답장 탭]
    ├─ 스타일 선택 → 답장 생성
    ├─ 무료 → 유료 전환 (화살 소모)
    └─ "내 정보 확인" → MY 탭
         ↓
[MY 탭]
    ├─ 큐피드 화살 보유량 확인
    ├─ 화살 구매 진입
    └─ 나의 연애 설명서 확인
```

### 1.3 핵심 UX 원칙

1. **감정 우선**: "읽기"보다 "판단"과 "행동" 유도
2. **단일 CTA**: 각 화면마다 명확한 하나의 행동 유도
3. **자연스러운 연결**: 다음 행동이 자동으로 떠오르게
4. **불안 제거**: "정답"이 아닌 "가능성" 제시

## 2. 탭별 연결 시나리오

### 2.1 홈 → 대화분석 → 답장 → MY

#### 시나리오 A: "오늘 뭔가 이상해"
```
홈 탭
  └─ "오늘, 이대로 가도 될까?" 카드 클릭
      └─ 연애 상세 페이지
          └─ "상대 마음이 궁금해" → 대화분석 탭
              └─ 분석 결과 확인
                  └─ "답장이 필요해" → 답장 탭
                      └─ 답장 생성 후 복사
                          └─ "화살 확인" → MY 탭
```

#### 시나리오 B: "답장이 급해"
```
홈 탭
  └─ 바로 답장 탭으로 이동
      └─ 답장 생성
          └─ 화살 부족 → MY 탭 (구매)
```

#### 시나리오 C: "내 연애 패턴이 궁금해"
```
홈 탭
  └─ MY 탭으로 이동
      └─ 반복 패턴 인사이트 확인
          └─ 심화 인사이트 (화살 사용)
              └─ "오늘 상태 확인" → 홈 탭
```

### 2.2 탭 간 연결 포인트

| 출발 탭 | 도착 탭 | 연결 트리거 | 카피 예시 |
|---------|---------|------------|-----------|
| 홈 | 대화분석 | 홈 하단 카드 | "상대 마음이 궁금해요" |
| 홈 | 답장 | 홈 하단 카드 | "지금 답장이 필요해요" |
| 대화분석 | 답장 | 분석 결과 하단 | "이 분석으로 답장 만들기" |
| 답장 | MY | 화살 부족 시 | "화살 충전하러 가기" |
| MY | 홈 | 화살 충전 후 | "오늘의 연애 확인하기" |

### 3.2 메인 카드 UI 스펙

#### 레이아웃
- **카드 크기**: 전체 너비, 높이: 약 280px
- **배경**: `todayMode.color.bg` 그라데이션
- **패딩**: 24px (p-6)
- **보더**: `border-white/50`
- **그림자**: `shadow-lg`

#### 요소 배치 (위 → 아래)

1. **날짜** (상단 좌측)
   - 폰트: `text-xs font-medium`
   - 색상: `text-black/70`
   - 예시: "12월 25일 (수)"

2. **상태 태그** (날짜 아래)
   - 배지 형태: `rounded-full px-3 py-1.5`
   - 배경: `todayMode.color.accent`
   - 텍스트: `{todayMode.modeEmoji} {todayMode.modeName}`
   - 예시: "🔥 급발진", "⚠ 거리 유지"

3. **핵심 한 줄 메시지** (태그 아래)
   - 폰트: `text-base font-semibold`
   - 색상: `text-black`
   - 줄간격: `leading-relaxed`
   - 예시: "오늘은 조금 조심스럽게 다가가는 게 좋아 보여요"

4. **보조 가이드** (선택, 메시지 아래)
   - 폰트: `text-sm`
   - 색상: `text-black/80`
   - 예시: "서서히 신뢰를 쌓아가요"

5. **구분선** (보조 가이드 아래)
   - `border-t border-black/20 pt-4`

6. **메인 CTA** (하단)
   - 레이아웃: `flex items-center justify-between`
   - 텍스트: `text-sm font-medium text-black`
   - 화살표: `text-black/70`
   - 고정 문구: **"오늘, 이대로 가도 될까? →"**

## 5. DAU·체류·과금 연결 전략

### 5.1 DAU (Daily Active Users) 전략

#### 홈 탭의 매력
- **매일 바뀌는 콘텐츠**: "오늘의 연애" 상태가 매일 갱신
- **감정적 질문**: "오늘, 이대로 가도 될까?" → 호기심 유발
- **단순한 구조**: 복잡하지 않아 부담 없음

#### 사용자 행동 패턴
```
매일 아침/저녁
  └─ 앱 열기
      └─ 홈 탭 확인
          └─ "오늘의 연애" 카드 클릭
              └─ 상세 페이지 확인
                  └─ 필요 시 대화분석/답장 사용
```

### 5.2 체류 시간 전략

#### 자연스러운 탐색 흐름
1. **홈 → 연애 상세**: 2-3분
2. **연애 상세 → 대화분석**: 5-10분 (분석 + 결과 확인)
3. **대화분석 → 답장**: 3-5분 (답장 생성)
4. **답장 → MY**: 1-2분 (화살 확인)

**총 체류 시간**: 약 11-20분

#### 체류 시간을 늘리는 요소
- **대화분석**: 심화 인사이트 (화살 잠금) → 호기심 유발
- **답장**: 여러 스타일 시도 → 재생성 버튼
- **MY**: 반복 패턴 인사이트 → 자기 이해

### 5.3 과금 전환 전략

#### 무료 → 유료 전환 포인트

1. **대화분석 탭**
   - 무료: 기본 분석 결과
   - 유료: "지금 쓰면 좋은 말 / 피해야 할 말" (화살 1개)
   - 유료: "단기 관계 전망" (화살 1개)

2. **답장 탭**
   - 무료: 하루 3회
   - 유료: 4회 이상 (화살 1개/회)

3. **MY 탭**
   - 무료: 기본 인사이트
   - 유료: 심화 인사이트 (화살 소모)

#### 과금 유도 메커니즘

**A. 자연스러운 막힘**
```
답장 생성 (무료 3회 사용)
  └─ "오늘 무료 답장을 다 썼어요"
      └─ "화살로 계속 이용하거나 내일 다시 만나요"
          └─ 화살 잔액 확인 (부족)
              └─ "화살 충전하러 가기" → MY 탭
```

**B. 호기심 유발**
```
대화분석 결과 확인
  └─ "지금 쓰면 좋은 말" (블러 처리)
      └─ "화살 1개로 열기"
          └─ 화살 잔액 확인 (부족)
              └─ "화살 충전하러 가기" → MY 탭
```

**C. 즉시 사용 가능**
```
화살 충전 완료
  └─ "지금 바로 사용하기" 버튼
      └─ 이전 탭으로 복귀 (답장/대화분석)
```

### 5.4 DAU → 행동 → 과금 연결 고리

```
[DAU]
  ↓
홈 탭 진입
  ↓
"오늘, 이대로 가도 될까?" 클릭
  ↓
연애 상세 확인
  ↓
[행동]
  ├─ "상대 마음이 궁금해" → 대화분석
  │   └─ 분석 결과 확인
  │       └─ 심화 인사이트 (화살 필요)
  │
  └─ "답장이 필요해" → 답장
      └─ 답장 생성 (무료 소진)
          └─ 화살 필요
              ↓
[과금]
  └─ MY 탭 → 화살 구매
      └─ 즉시 사용 가능
```

### 5.5 핵심 성공 지표 (KPI)

1. **DAU**: 홈 탭 일일 방문 횟수
2. **체류 시간**: 탭 간 이동 횟수
3. **과금 전환율**: 무료 사용 → 화살 구매 비율
4. **재방문율**: 다음 날 다시 방문하는 비율
